Email Opened: Removing the UI
  Ironically, my best idea for Email Opened was cross-pollinated by Basecamp Breeze. Almost immediately after I read the launch announcement of Breeze, it hit me, "Why does Email Opened even need a web UI at all?" People already know how to write an email to their best friend or spouse. What if sending a blast to 1,000 people was as easy as writing an email to one person?
  
  To flesh out the idea, our customers could select a list ID (eg. soccer-team, my-church). They could then send their message to a special email address (soccer-team@list.emailopened.com) using their current email client. We would process their message and send it to their contacts.
  
  What could be better than an easy UI? No UI.

DalJS: Web Apps without PHP, Ruby, JS, Python
  This was an ambitious, tongue-in-cheek, weekend project to rid the world from all server-side architecture. The goal was to write a functioning web app using only HTML. First, an included javascript file would parse the coder's intentions, then connect with a backend where permissions were preset for each model. It meant shaking up HTML standards a bit, and using things like this:

  - <form for="Campaign" notify="members">
  - <ul collection="Campaigns">
  - <a href="#" show="Campaign at #main">

  DalJS was an interesting project that proves I'm ready to test all assumptions. Solution-finders are willing to experiment, question, and stay un-married to the past.

LiveHours.co: Customer-Driven Development
  LiveHours.co launched because of a request from a realtor friend. He wanted a "Chat Now" button for his website. As a small, one-man business, he couldn't afford to sit in front of his computer all day, waiting to respond to incoming chat messages.

  I could have created an app for his iPhone and Android devices, but I decided on a more ubiquitous approach: SMS. My friend already knew how to text message his friends. What if he was able to reply to visitors on his website just as easily?
  
  Now, he can respond to chat requests on his site from anywhere - simply by replying to the text message!
  
  No UI necessary.
